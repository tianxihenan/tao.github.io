实现一个优先级队列

  1. 首先定义了一个名为PriorityQueue的类，初始化方法__init__中创建了一个空列表_queue用于存储队列中的元素，并设置一个索引变量_index初始化为0，该变量用于在堆中插入元素时确保即使优先级相同也能保持元素的顺序。

  2. 类中有一个push方法，接受两个参数：item（要插入队列的元素）和priority（该元素的优先级）。这里使用heapq库的heappush函数将元素以元组形式（-priority, _index, item）插入到_queue中。注意，因为Python的heapq是小顶堆，所以我们取-priority来实现大顶堆的效果。每插入一个元素后，索引_index递增1，以便于区分优先级相同的元素的顺序。

  3. pop方法用于从优先级队列中移除并返回优先级最高的元素。这里使用heapq库的heappop函数弹出最小值（即实际的最大优先级），然后返回这个元组的最后一个元素，即原始插入的item对象。

  4. 定义了一个简单的Item类，用于创建队列中的元素，其构造函数接受一个name参数，并在打印或表示时返回该name值。

  5. 创建了一个PriorityQueue实例q，并向其中插入了四个Item对象，它们的优先级分别为1、5、4和1。

  6. 最后，通过循环从优先级队列q中弹出并打印4个元素。由于队列按照优先级排序，所以输出顺序将是按优先级从高到低排列的Item对象。

  7. 在上述优先级队列实现中，我们看到 -priority 的使用是为了构建一个“最大堆”。

    通常，Python 的 heapq 模块实现的是一个最小堆，即堆顶元素总是所有元素中最小的那个。但在优先级队列的应用场景下，我们往往需要根据优先级的大小来决定元素的出队顺序，优先级越高的元素应该越先出队。

    为了利用 heapq 实现一个优先级高的元素先出队的优先级队列，这里采用了一种技巧：将优先级取负数再插入堆中。这样，原来优先级高的元素经过变换后，在堆中表现出来的值反而小（因为是负数），因此在堆顶得到的是“最小”的那个元素，实际上对应的是原数据中优先级最高的元素。

    例如，若有两个元素，优先级分别为 1 和 5，则在堆中插入时，它们会变为 -1 和 -5，-5 显然小于 -1，因此 -5 对应的元素会在堆顶，也就是最先出队，符合优先级队列的设计初衷。

heapq.heappush是Python标准库heapq模块中提供的一个函数，它用于将一个元素添加到已存在的堆结构中，同时保证堆的性质——即堆顶元素始终是最小（对于最小堆）或最大（对于最大堆，通常通过调整元素的方式实现，如这里是负数优先级实现大顶堆）的元素。

函数签名如下：

  • heap：是一个列表，表示当前的堆结构。在Python中，堆是通过一个普通的列表来实现的。
  • item：是要插入堆中的新元素。

当调用heappush(heap, item)时，函数会将item添加到heap列表的末尾，然后通过下沉操作（sift down）确保新插入的元素不会破坏堆的有序特性。这意味着如果item比它的父节点小（对于最小堆），那么它会被与其父节点交换位置，这一过程可能递归地继续下去，直到item处于正确的位置，使得整个堆再次满足堆的性质。

set

在Python编程语言中，set 是一种内建的数据结构，它代表了一个无序且不重复的元素集合。这意味着当你向 set 中添加元素时，它会自动去除重复项，并且集合内的元素没有固定的顺序。你可以把它想象成一个“唯一值容器”。

set 的主要特点包括：

  1. 无序性：集合内的元素没有特定的顺序，所以无法通过索引访问某个元素，也不能保证每次迭代或打印集合时元素的顺序相同。

  2. 唯一性：集合内的元素是唯一的，即使你尝试添加相同的元素多次，集合中也只会保留一个实例。

  3. 可变性：集合是可变的，可以添加新的元素、删除元素以及执行集合间的数学运算，如并集（union）、交集（intersection）、差集（difference）和对称差集（symmetric difference）等。

  4. 元素要求：集合中的元素必须是不可变的，即它们不能是列表、字典或其他可变类型，但可以是整数、浮点数、字符串、元组或者其他不可变类型。

NameError（名称错误）：
NameError 发生在 Python 解释器尝试访问一个尚未被定义（即不存在）的变量、函数、类或其他标识符时。这意味着你在代码中使用了一个名字，但是 Python 在当前作用域内找不到与这个名字关联的任何有效对象。常见的导致 NameError 的情况包括：

3. import与from的区别

  • 使用import语句导入整个模块时，我们需要使用模块名来访问其中的函数、类和变量。例如，module_name.function_name()。
  • 使用from语句导入特定函数、类或变量时，我们可以直接使用导入的名称，而无需使用模块名。例如，function_name()。
